Relational inductive biases, deep learning, and graph networks

Nov 05, 2022 in Paper Notes

Introduction

Combinatorial generalization is constructing new inferences, predictions, and behaviors from known building blocks. Here we explore how to improve modern AI‚Äôs capacity for combinatorial generalization by biasing learning towards structured representations and computations, and in particular, systems that operate on graphs.

Relational inductive bias

üìå Box 1: Relational reasoning

  • An entity is an element with attributes
  • A relation is a property between entities
  • A rule is a function that maps entities and relations to other entities and relations

üìå Box 2: Inductive biases

An inductive bias allows a learning algorithm to prioritize one solution (or interpretation) over another.

In deep learning, the entities and relations are typically expressed as distributed representations, and the rules as neural network function approximators.

Relational inductive biases in standard deep learning building blocks

Fully connected layers

The implicit relational inductive bias in a fully connected layer is thus very weak: all input units can interact to determine any output unit‚Äôs value, independently across outputs

Convolutional layers

There are some important relational inductive bias in convolutional layers: locality and translation invariance.

Recurrent layers

The rule which takes a step‚Äôs inputs and hidden state as arguments to update the hidden state, is reused over each step, which reflects the relational inductive bias of temporal invariance.

Relational inductive bias in graph networks

First, graphs can express arbitrary relationships among entities. Second, graphs represent entities and their relations as sets, which are invariant to permutations. Third, a GN‚Äôs per-edge and per-node functions are reused across all edges and nodes, respectively.

Design principles for graph network architectures

Flexible representations

Graph networks support highly flexible graph representations in two ways

  • the representation of the attributes
  • the structure of the graph itself

Attributes

The requirements of the problem will often determine what representations should be used for the attributes. The output of a GN block can also be tailored to the demands of the task.

Graph structure

When defining how the input data will be represented as a graph, there are generally two scenarios:

  • the input explicitly specifies the relational structure
  • the relational structure must be inferred or assumed

Composable multi-block architecture

A key design principle of graph networks is constructing complex architectures by composing GN blocks.

Composition of GN blocks

Arbitrary numbers of GN blocks can be composed.

  • The blocks can be unshared (different functions and/or parameters, analogous to layers of a CNN),
  • or shared (reused functions and parameters, analogous to an unrolled RNN), . Shared configurations are analogous to message-passing where the same local update procedure is applied iteratively to propagate information across the structure. The information that a node has access to after m steps of propagation is determined by the set of nodes and edges that are at most m hops away

Encode-process-decode

For example, the encoder might compute the initial forces and interaction energies between the balls, the core might apply an elementary dynamics update, and the decoder might read out the final positions from the updated graph state.
